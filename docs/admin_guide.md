# Руководство по администрированию

## Мониторинг

### Логи
- Логи бота: `logs/app.log` — проверка ошибок Telegram API, ошибок БД, Redis.
- Логи Redis/БД (если используются внешние системы).

### Метрики
- Кол‑во обработанных join‑запросов за период.
- Количество успешных/отклонённых запросов.
- Среднее время ответа капчи.

Рекомендуется интеграция с Prometheus/Grafana или простая статистика по логам.

## Обработка блокировок Telegram

Если аккаунт бота или пользователя‑бота заблокирован:

1. Получи новый `BOT_TOKEN` у `@BotFather`.
2. Обнови `.env`.
3. Перезапусти бота.
4. Проверь, что:
   - БД корректно восстанавливается из бэкапа;
   - FSM‑состояния Redis не потерялись (если нужно — очистить/пересоздать).

## Резервное копирование и восстановление

1. Регулярно копируй:
   - `data/applications.db`
   - (опционально) Redis dump файл.
2. При восстановлении:
   - Останови бота.
   - Подмени старый `applications.db`.
   - Запусти бота.

## Обновления

- Обновление `aiogram`, `redis`, `SQLAlchemy` — всегда через `pip` внутри виртуального окружения.
- Проверка совместимости с Python 3.x в `pyproject.toml`/`requirements.txt`.
- Тестирование на тестовом боте перед продакшеном.
