# Характеристики качества ПО telegram-join-manager

## 1. Стабильность

### Описание
Система должна стабильно работать в течение длительного времени без падения из‑за внутренних ошибок.

### Показатели
- **MTBF (Mean Time Between Failures):** > 7 дней.
- **Среднее время восстановления (MTTR):** < 30 минут (перезапуск бота, восстановление БД из бэкапа).
- **Отказоустойчивость:** бот перезапускается при ошибке и корректно сохраняет состояние через Redis/БД.

### Требования
- Работа в фоновом режиме (долгий запуск `asyncio`‑цикл).
- Обработка сетевых ошибок от Telegram API (retry‑логика, логирование).
- Корректное завершение (`close_db`, закрытие Redis‑подключений).

---

## 2. Производительность

### Описание
Система должна обрабатывать join‑запросы с минимальными задержками.

### Показатели
- Время ответа капчи: < 10 секунд.
- Время принятия/отклонения запроса: < 5 секунд.
- Поддерживается до 1000 join‑запросов в час на один бот (при тестировании).

### Требования
- Оптимизация ORM‑запросов к БД.
- Использование асинхронных библиотек (`aiosqlite`, `aiogram`, Redis async).
- Отсутствие блокирующих операций в main loop.

---

## 3. Безопасность

### Описание
Обеспечение конфиденциальности и целостности данных пользователей.

### Показатели
- Хранение `bot_token`, `API_ID`, `API_HASH` в `.env`, без коммита в репозиторий.
- Шифрование/обфускация sensitive‑данных в БД, по возможности.
- Ограничение доступа к Redis/БД извне.

### Требования
- Валидация входящих данных (id, usernames, ссылки).
- Проверка источника запроса (только от Telegram).
- Регулярный аудит логов на признаки аномальных запросов.

---

## 4. Поддерживаемость

### Описание
Лёгкость внесения изменений, масштабирования и исправления ошибок.

### Показатели
- Наличие документации (`docs/`, README).
- Разделение ответственностей (bot, db, redis, config).
- Наличие тестов (unit, integration) и линтера (ruff, mypy).

### Требования
- Чистая архитектура модулей.
- Конфигурация через `pydantic-settings` и `.env`.
- Логирование событий в файл (`logs/app.log`).
